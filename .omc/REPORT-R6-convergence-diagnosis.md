```
最快   C1, C2     500 spp 以內乾淨
最慢   C3         收斂最慢
次慢   C4         收斂第二慢
```

## §3. 根因推斷

### 3.1 問題類型確認

這是**高變異數（high variance）問題**，不是**偏差（bias）問題**。

- 偏差問題的症狀：跑再多 SPP 結果仍然系統性偏移
- 變異數問題的症狀：跑夠多 SPP 後收斂到正確結果，但中途噪點高

實測 20,000 spp 乾淨 → 確認估計式是無偏的（unbiased），只是變異數過高。

### 3.2 C3 / C4 慢的原因

C4 特性：多光源 + 多反彈 + 多次直接光取樣  
C3 特性：最複雜燈光配置

共同點：**小面積、高亮度光源**（22W COB 投射燈 + 25W 廣角軌道燈）

高變異數的機制：

```
小燈面積 → 隨機光線極難恰好命中燈面 → 取樣機率極低
偶發命中 → 能量值爆高（螢火蟲）
需要大量 SPP 才能把偶發高值平均攤平
```

### 3.3 暗部螢火蟲的額外解釋

暗部路徑 = 多次反彈後光量已衰減，但若此時碰到小燈，相對能量比更高，
造成暗部螢火蟲比亮部更明顯、更難被均值壓掉。

## §4. 對 ROADMAP 假說的更新

```
假說 1（NEE PDF 對小光源不利）   → 機率高，C3/C4 小燈場景直接驗證方向
假說 2（NEE 效率本身不夠）       → 已包含於假說 1，不需獨立驗證
假說 3（OIDN 整合可行性）        → 優先序降低，根因已指向 in-renderer 改善
假說 4（ReSTIR 可行性）          → 仍需查，但現在有更明確動機（小燈 + 多燈）

H 診斷（幾何問題）               → 優先序降低（C1/C2 快速收斂排除幾何系統性問題）
G 診斷（NEE PDF 公式）           → 優先序提升，現為最高優先
```

---

## §5. 實驗建議更新

```
原建議   G + H 並行診斷
更新後   G 診斷優先，H 可降為備案

理由：C1/C2 用同一場景幾何卻快速收斂
     → 幾何問題若存在，應該也讓 C1/C2 慢下來
     → 既然 C1/C2 快，幾何問題可能性低
     → C3/C4 慢的原因更可能在「燈光取樣策略」而非「幾何」
```

---

## §6. 成功標準更新

```
原標準   1,000 spp 無螢火蟲、陰影雜點降到肉眼可接受
更新後   C3 / C4 在 1,000 spp 以內達到現在 C1/C2 的 500 spp 水準
         亦即：目標是讓 C3/C4 收斂速度追上 C1/C2，而非要求所有 config 同時達標
```
