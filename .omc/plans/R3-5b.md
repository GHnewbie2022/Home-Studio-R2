Policy (a) 保守: Cloud 入 stochastic NEE pool；BSDF-indirect to Cloud 不累加；MIS 留 R3-6。

## Principles (RALPLAN-DR)
1. **三源同值原則**: NEE 選面數 = hit-branch emissive 面數 = JS Φ 分母，三處以同一常數 `CLOUD_FACE_COUNT = 2` 表達（甲案）。
2. **Pattern 繼承**: 同構 R3-5a sampleStochasticLight7 → `sampleStochasticLight11`；rod → target sampling 延續 R3-4 fix04（射面中心，非 pivot）。
3. **Throw-first 守門**: 新增 `uCloudObjIdBase` / `CLOUD_FACE_COUNT` mismatch assertion，模仿 TRACK_LAMP_ID_BASE pattern。
4. **量綱對齊**: 保留 Φ/(K·π·A) radiometric；rod 對其能量損失用 lumens 推導地板 Ev 量化。
5. **R3-6 Forward-compat**: L1397 本階段清洗為 `bounceIsSpecular == TRUE`，避免 MIS 階段復活雙累加。

## Decision Drivers (top 3)
1. **正確性優先**: 三源不自洽會致 throughput 系統性偏差（iter1 1.5× 被 Architect 擋）。
2. **R3-3 既存污染同 commit 清洗**: L482 /3、L195 註解、L1387 isEmissiveFace 全部對齊；executor 不得拆 commit（Critic C1）。
3. **可驗證性絕對化**: AC 數值閾值用絕對 cd/m² + lumens 推導，非主觀百分比。

**CLOUD_FACE_COUNT = 2** 為全文常數，三源引用同值。

## ADR

- **Decision**: Cloud 以 2-face (甲案) 進 stochastic NEE pool，pool size 7→11，selectPdf=1/11，face pick 1/2。
- **Drivers**: 正確性（三源同值）、簡單性（face 等面積）、forward-compat（R3-6 MIS 可升 4 面）。
- **Alternatives considered**:
  - 乙案（4 面不等面積 + area-weighted pick）: 複雜度高，uniform footprint +2；R3-5 過度設計。
  - 保留 `|| sampleLight`: R3-6 MIS 階段雙累加風險；清洗成本為零。
- **Why chosen**: Architect + Critic 共識；能量損失 1.3% 量化可接受；executor 實作路徑最短。
- **Consequences**:
  - 短端兩面合計 0.66% 能量損失（見附錄 A 計算）
  - L1397 `|| sampleLight` 本階段刪除，R3-6 MIS 若需 emissive hit-branch 直接用純 `bounceIsSpecular == TRUE`
  - uniform footprint +2 陣列（uCloudRodCenter[4]、uCloudRodHalfExtent[4]）
- **Follow-ups**:
  - R3-6 MIS Cloud 入 balance/power heuristic
  - 若視覺需求顯示短端貢獻明顯，R3-6+ 升 4 面（改 ADR 不破 R3-5b）

## 附錄 A: 能量損失計算

單 rod（E/W 長 2.4m）: 總面積 = 2(頂+底) × 0.016×2.4 + 2(長側) × 0.016×2.4 + 2(短端) × 0.016×0.016
 = 2×0.0384 + 2×0.0384 + 2×0.000256 = 0.1541 m²
2-face emissive (頂+外長) = 0.0768 m²；短端 = 5.12e-4 m²
短端 / (2-face + 短端) = 5.12e-4 / 0.0773 ≈ 0.66% per rod；4 rod 同構 → 總能量相對完整 4-face (含短端) 損失 ≈ 0.66%。

註: iter1 誤算為 1.3%（當時估 0.65% × 2 短端雙計），實為 0.66%。ADR Consequences 行據此更正。
