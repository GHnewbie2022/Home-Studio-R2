## 2. a2 偏離評估（ceiling always-on）

Planner 選 Critic option A 之 a2（ceiling 排除 Principle 3，不新增 `uCeilingEnabled`）而非 a1（補 uniform）。此決策評估：

- **技術正確性**：grep shader 現無 `uCeilingEnabled`，a2 反映實際 code state，無虛構 uniform 風險。AC-M6 「`grep uCeilingEnabled` 確認不存在」為守門。
- **ADR 責任鏈完整**：L185 Consequences「若需 disable pathway，另開 R3-8 補 `uCeilingEnabled`」明記技術債，使用者採購評估或晚間情境測試需求觸發即升 R3-8。
- **合理性判斷**：本專案為 **錄音室場景**（Home_Studio），ceiling 為「房間基礎照明」語意強烈，R3-6 停機不開 ceiling 無商業情境（使用者為音訊工程工作流程，非戲劇燈光展示）。R3-8 採購評估若需單燈比對（喇叭顏色於冷光 vs 暖光），屆時補 uniform 亦合理，不影響 R3-6 結構。
- **結論**：a2 屬 **合理技術債**，非 blocker。a1 會迫使 executor 中途新增 uniform + JS hook + GUI checkbox（scope creep），a2 簡潔且責任鏈明確。

## 3. 新架構風險（iter 2 修訂引入）

### N1. R4 state-clear 覆蓋完整性（輕微，非 blocker）

Plan L112 列 7 處 `bounceIsSpecular = FALSE` 對應行號 L929/L970/L1007/L1041/L1060/L1110/L1141。無法於 plan 階段逐行驗 main loop 每條 DIFF→SPEC 轉換是否皆含於此 7 行內——L1110 / L1141 於 R3-5b 實際 shader 為「hitType==CLOUD_LIGHT 後分支終結」或「loop bottom」待 executor pre-flight 確認。I4 integration test（L81 注入 `misWPrimaryNeeLast = 42.0`）為良好守門，但 executor 落地時建議再 grep 一次 `bounceIsSpecular\s*=\s*FALSE` 與 `bounceIsSpecular\s*=\s*TRUE` 兩側對稱檢查。**不列為 blocker**（I4 已涵蓋 runtime 驗證）。

### N2. uR3MisPickMode 與 uR3MisEnabled 無 uniform 衝突（通過）

兩 uniform 皆為 float 獨立 slot（L137「`uR3MisEnabled = { value: 1.0 }` + `uR3MisPickMode = { value: 0.0 }`」），命名前綴一致、語意正交（MisEnabled = 全局 gate，MisPickMode = pick 策略），無 slot budget 問題（vec4 占用 +2 float = <1 vec4）。R3-7 升乙案僅需翻 `uR3MisPickMode = 1.0` 並 fill `uR3LightPickCdf`，shader 簽名不破。**通過**。

### N3. a2 pool idx 0 gate 內部一致性（通過）

NEE pool idx 0 為 ceiling quadLight，`sampleStochasticLight11` 於 idx 0 無 GUI gate 需求（ceiling always-on），其他 10 盞各自於 NEE 內部或 pool 外部由 primary-hit + NEE + BSDF-hit 三處守門。a2 選擇不留「idx 0 silently skipped」漏洞——L109「`uR3MisEnabled < 0.5` gate bypass 回 R3-5b 路徑。**無 `uCeilingEnabled` gate**」明確。**無 gate inconsistency**。

### N4. AC-V1 FWHM 測量仍含主觀成分（輕微，非 blocker）

AC-V1 L166 operationalize 為「FWHM ≤ 2× baseline width，或 binary 判可辨形狀（圓 / 橢圓 / 方塊）」。FWHM 測量需 executor 於 Cam 2 876 spp 截圖上用 ImageJ 或 Photoshop 取像素 line profile，相對「完全主觀肉眼」已具體化但仍有測量誤差（±1 pixel 於 2× 判定下 ~5% 誤差）。Binary fallback「可辨形狀」為主觀 escape hatch，若 FWHM 測量邊界值出現爭議，binary 判定具決定性。**屬可接受折衷**（完全客觀需 MSE/SSIM 對 reference ground truth，本階段代價過高）。
