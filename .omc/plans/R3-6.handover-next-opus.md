## 第 2 步：R3-6.5 動態光源池（ralplan）

### 根因分析（使用者觀察 + 認可）
R2 baseline 只有吸頂燈時底噪乾淨，R3-5a ~ R3-6 之後底噪明顯變大。

真正原因：`sampleStochasticLight11` 寫死 11 格 uniform stochastic pick。
CONFIG 1 / CONFIG 2 下實際只有吸頂燈啟用（Cloud / Track / Wide 皆 disabled）
→ 11 次擲骰子中 10 次擲到關閉的光源
→ helper 回傳 `throughput = 0` 直接丟棄
→ 有效樣本僅 1/11
→ 理論底噪 √11 ≈ 3.3 倍惡化

使用者明確期望：CONFIG 1 / 2 底噪應回到 R2 純吸頂燈基準。

### 建議技術方向
- 每 frame JS 端計算啟用光源數 N，傳入 uniform `uActiveLightCount`
- 建立 enabled light index LUT uniform，或每 pool 重新緊縮索引
- helper 改名 `sampleStochasticLightDynamic`，`rng() × N`，`selectPdf = 1 / N`
- N = 1（CONFIG 1 / 2 純吸頂燈）時退化為 sampleQuadLight 單一光源路徑，需驗證 byte-equivalent
- reverse-NEE Probability Density Function 與 Multiple Importance Sampling heuristic 連動校準（helper `pdfNeeForLight` 內 `1.0/11.0` 全數改 `1.0/uActiveLightCount`）

### 風險（ralplan 需 deliberate 模式）
- 動 `selectPdf` = 動所有光源的 Probability Density Function（機率密度函數），連帶 MIS power heuristic 權重改變
- checkbox 動態 toggle 時需即時重建 index LUT，不可中斷渲染累積
- Phase 1 仍須保留 uR3MisEnabled rollback（關閉回 R3-6 11-slot 路徑）以利二分定位
