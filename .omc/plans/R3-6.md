# R3-6 MIS Phase-1（ceiling + Cloud 5 DIFF emitters）— Plan (iter 2, DELIBERATE)

Policy (A') 保守 MIS，**scope 縮至 ceiling quadLight + Cloud 4 rod = 5 DIFF-emitters**。對此 5 盞 DIFF 類表面之 NEE 與 BSDF-sample 雙路徑施加 **power heuristic β=2**；Track/Wide 6 盞 cone 光本階段 **BSDF-hit emission 維持 R3-5b `bounceIsSpecular == TRUE` 直接累加路徑**，不套 MIS（避免 cone 外能量洩漏，詳 Principle 6）。SPEC/REFR 不套 MIS，delta BSDF 直接累加。全流程以 `uR3MisEnabled` uniform 閘門，0 = 回落 R3-5b 肉眼等價路徑（pixel diff < 2/255, 99th percentile）。R3-7 補 cone-aware NEE PDF 後再擴 MIS 至 Track/Wide。

## Principles (RALPLAN-DR)

1. **Baked-emission 契約延續**：`sampleStochasticLight11` 回傳之 `throughput` 已把 `emission × geom × A` 烘入；MIS 權重只除不乘、不得再乘 emission。對齊 R3-5b L243/L269/L309 pattern。**Scope：ceiling quadLight + Cloud 4 rod，共 5 DIFF-emitters**。
2. **PDF 量綱一致**：`p_nee` 與 `p_bsdf` 必須同處 solid-angle domain。`p_ω = p_area · dist² / cos_light`，否則 heuristic 分母無因次錯值。
3. **雙源同結構 gate（R3-5b fix07 延續）**：Cloud 4 rod 之 `uCloudLightEnabled` 於 primary-hit + NEE pool + 新增 BSDF-hit MIS 分支三處對稱 gate；Track/Wide 6 盞之 `uTrackLightEnabled` / `uWideTrackLightEnabled` 維持 R3-5b 之 primary-hit + NEE pool 兩處 gate（本階段無 BSDF-hit MIS 分支，仍以 grep 驗 ≥ 2 次守門）。**Ceiling quadLight 例外**：ceiling 為房間基礎照明，現行 shader 無 `uCeilingEnabled` uniform（grep 確認），於 R3-6 明確定位為 always-on 主燈，僅受 `uR3MisEnabled` 全局開關影響，不納入三源 gate 規則（ADR Consequences 明記）。
4. **Cone-falloff 定位不變**：Track / Wide 之 cone smoothstep 僅於 NEE 內部烘入 throughput（L242 / L265）；primary-hit emission 採 `uTrackEmission[i]` 無 cone（對齊 Cloud `min(emission, clamp)` 直取）。R3-6 不動此契約，亦不於 BSDF-hit 套 MIS（見 Principle 6）。
5. **Rollback-by-uniform（肉眼等價）**：`uR3MisEnabled.value = 0` 時 shader 走 R3-5b 原 code path，pixel diff < 2/255 (99th percentile) + mean abs diff < 0.3/255（放寬自 iter 1 「byte-identical」，因 `sampleStochasticLight11` 簽名 4→6 args 會觸發 GLSL fma 重排 sub-LSB drift，byte-identical 數學上不可達）。
6. **Cone 光 MIS 延後 R3-7**：Track / Wide 6 盞之 cone falloff 為 NEE-only 契約。若於 MIS BSDF-hit 分支累加 emission，NEE throughput 雖於 cone 外 baked 為 0，但 `p_nee` solid-angle PDF 仍為正值；heuristic `w_bsdf = p_bsdf² / (p_nee² + p_bsdf²)` 不會於 cone 外歸零，導致 BSDF 路徑對 cone 外 rawEmit 多加一份能量（C1 成案）。R3-6 明確排除此 scope，待 R3-7 落地 cone-aware NEE PDF（即 `p_nee` 亦 bake cone falloff 為 0）後再擴。

## Decision Drivers (top 3)

1. **雜訊降低可量測**：Cam 3 876 spp 地板中心 Lv 跨 3 次 render 標準差相對 R3-5b baseline 下降 **≥ 30%**。
2. **金屬鏡射銳利化可辨**：Cam 2 喇叭前面板對 **ceiling quadLight** 之鏡射高光 FWHM 寬度 ≤ 2× R3-5b baseline width，或 binary 判定可辨形狀為「圓點 / 橢圓 / 方塊」非混糊光暈（R3-5b baseline reference screenshot 876 spp 附於 `.omc/plans/R3-6.ref.png`）。
3. **量綱零回退 + R3-5b rollback 完整**：Φ/(K·π·A) 三燈類契約完全不動；gate=0 路徑以 grep + pixel diff 雙驗。

## Viable Options

### 甲（採納）: Power-heuristic β=2 + uniform 1/11 light-pick + ceiling + Cloud 5 DIFF-emitters

- Light-pick PDF：`selectPdf = 1/11`（保守，延續 R3-5b pool；Track/Wide 仍於 pool 內被抽，僅其 BSDF-hit 分支不套 MIS）。
- NEE 路徑：`sampleStochasticLight11` 新簽名 `out float pdfNeeOmega, out int pickedIdx`；主 loop 以 `misWPrimaryNeeLast` 跨 bounce 攜 PDF state。
- BSDF 路徑：僅 ceiling LIGHT 與 Cloud rod 於 BSDF-indirect 命中時計 `p_bsdf = cos(θ)/π`，heuristic β=2。
- SPEC / Track LIGHT / Wide LIGHT 於 BSDF-indirect：延續 R3-5b `bounceIsSpecular == TRUE` 直接累加；shader 簽名更新後，此分支**不讀** `misWPrimaryNeeLast`。
- **Pros**：C1 能量洩漏完全消除；實作路徑最短；Cam 3 主 use case（ceiling-dominated）BSDF 權重 well-balanced 於近距 ceiling hit 可收 30-50% variance 降噪。
- **Cons**：MIS 收益集中於 ceiling + Cloud 子集；Track/Wide 降噪留 R3-7；uniform pick 忽略 power 差異（詳 Alternatives）。

### 乙（不採此階段，保留為 early-exit fallback）: Power-proportional CDF pick + MIS

- 每 frame 以 emitter flux 為權重 build 11-slot CDF（JS 端 prefix-sum），shader binary-search。
- **誠實 projected MSE gain on this project: 40-70%**（非 iter 1 援引之 Cornell <15% baseline），依據 Veach thesis Fig 9.3 於 509× emitter area span 下之實測（附錄 B 自證：p_ω ceiling 0.0631 vs Track 32.1 = 509× span，iff uniform pick）。
- **Cons**：JS 端 GUI 調光即 CDF 重算，雙源同步風險；binary-search shader 指令數 +15%；與 R3-6 scope（MIS 架構正確性）正交，不應混同。
- **本階段 reject 理由**：(a) 實作最簡原則下甲案已達 AC-V1 + AC-N3；(b) Cam 3 主 use case 為 ceiling-dominated 處 BSDF 權重 well-balanced；(c) **若 AC-V1 fail → 乙為 canonical next step，非跳 R3-7 R3-8**（ADR early-exit 明記）。

### 丙（不採）: LTC / ReSTIR DI

實作成本高，本專案 11 盞未達經濟點。明確 rejected，不排期。

**選甲理由**：Architect R1 + Critic C1 共識——cone 光能量洩漏正確性問題於 Track/Wide MIS 下必然發生，scope 縮至 5 DIFF-emitters 後 variance 降噪與金屬鏡射銳利化收益仍達標；乙案保留為 AC-V1 fail 時之 canonical fallback。

## ADR

- **Decision**：採 Policy (A') 保守 MIS——uniform 1/11 pick + power heuristic β=2，**scope 限 ceiling quadLight + Cloud 4 rod（5 DIFF-emitters）**；Track/Wide 6 盞 BSDF-hit emission 維持 R3-5b 不累加，延後 R3-7 cone-aware NEE PDF。SPEC/REFR 不套 MIS。Cloud rod 45° 對角 face 新 const `CLOUD_DIAGONAL_FACE_AREA[4]` 專供 PDF。`p*p` (非 `pow(p, 2.0)`) 實作 β=2 power：grep shader 現無 `pow(x, 2.0)` 實例，Mesa/Apple-M fp32 `pow(x, 2.0)` 走 `exp2(log2(x)*2)` 於小 x 有 ~1 ULP 精度損失，`p*p` 單指令 fma <0.5 ULP。
- **Drivers**：variance 降噪可量測（AC-N3）、金屬鏡射銳利化 operationalized（AC-V1 FWHM）、量綱零回退、R3-5b rollback 肉眼等價。
- **Alternatives considered**：

## 附錄 A：Power-heuristic β=2 公式導出

對 light source 積分 $\int_{\Omega} L_e(\omega) f_r(\omega) \cos\theta \, d\omega$，β=2 power heuristic：

$$
w_{nee}(\omega) = \frac{p_{nee}(\omega)^2}{p_{nee}(\omega)^2 + p_{bsdf}(\omega)^2}, \quad w_{bsdf}(\omega) = \frac{p_{bsdf}(\omega)^2}{p_{nee}(\omega)^2 + p_{bsdf}(\omega)^2}
$$

$w_{nee} + w_{bsdf} = 1$ 恆成立（AC-N4 守門）。本 shader 於 N 次 NEE + M 次 BSDF 採樣之 sum estimator 中，**n_nee = 1、n_bsdf = 1**（單次 shadow ray + 單次 BSDF ray per bounce），故估計子退化為：

$$
\hat{I} = w_{nee}(\omega_{nee}) \frac{L_e f_r \cos\theta}{p_{nee}(\omega_{nee})} + w_{bsdf}(\omega_{bsdf}) \frac{L_e f_r \cos\theta}{p_{bsdf}(\omega_{bsdf})}
$$

β=2 相對 β=1（balance）優勢於 p_nee ↔ p_bsdf unequal 時抑制 outlier；Veach thesis 於 high dynamic range light scene 實測全局 MSE 減 5-15%（specific test scenes，非 universal）。
