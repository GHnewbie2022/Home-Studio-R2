## 六、舊專案已有的單位換算函式（R3 必須搬入）

檔案：`/Users/eajrockmacmini/Documents/VS Code/My Project/Home Studio 3D Pace Tracing/Path Tracking 260412a 5.4 Clarity.html:1752-1755`

```javascript
function lumenToCandela(lm, beamAngleDeg) {
    const halfRad = Math.max(0.01, (beamAngleDeg / 2.0)) * Math.PI / 180.0;
    return lm / (2.0 * Math.PI * (1.0 - Math.cos(halfRad)));
}
```

**物理根據**：spot light 在 beam cone 內光度均勻假設下，candela = lumens / solid_angle，立體角 Ω = 2π(1 - cos(θ_half))。

**R3 shader 端 radiance 換算**：

```
L (W/sr/m²) = I (cd) / A (m²) × (683 × V(λ))^-1 × spectral_factor
簡化版（假設 photopic 中心波長）：
L_RGB = I_cd × colorTempToRGB(K) / (emitterArea × 683)
```

Track 投射燈 emitter 面積：π × 0.03² ≈ 2.827e-3 m²（圓柱端面 r=3cm）
Track 廣角燈 emitter 面積：π × 0.05² ≈ 7.854e-3 m²（圓柱端面 r=5cm）
Cloud 燈條 emitter 面積：每根燈條側出光面總面積（依長度 × 16mm）

## 八、色溫 → RGB 換算（R3 新增）

**目標**：每個光源獨立色溫，shader 端從 Kelvin 算 RGB，不再用手調 WARM/NEUTRAL/COOL。

**演算法選擇**：Tanner Helland 近似（快速，夠準）或 Planck 黑體 → CIE XYZ → sRGB（慢但正確）。

**建議色溫配置**：

```
Cloud 間接燈（漫射氛圍）：4000K（柔和）
軌道投射燈（主燈）    ：4000K（工作）
軌道廣角燈（補光）    ：4000K（避免色偏）
→ 全室統一 4000K，對應舊專案 colorTemp 預設
```

若使用者想模擬其他情境（夜間 3000K / 日光 6500K），可全室一起換。

## 附錄：為何只搬數字不搬架構

舊專案 `Path Tracking 260412a 5.4 Clarity.html` 的 shader 架構有這些風險：

- 光源直接寫死進 sceneIntersect，與 R2-18 的 fixtureGroup gating 不相容
- NEE 權重計算用 per-light wTx 變數（L1217-1224）而非 light array + stochastic selection，光源超過 10 盞時會爆炸
- 無 MIS，只有 NEE，高光反射在粗糙表面上會 variance 爆炸
- `mask *= 1.5` 在舊專案沒有，但現專案 R2-18 有 10 處（L837~1255），是 Phase 2 補償

所以 R3 照計畫：**架構沿用 R2-18 + Many-Light Sampling + MIS power heuristic**，**數字搬自本文件**。
