## 資料來源

### 幾何架構說明
舊專案的 `archBoxes` / `objConfig` 是舊專案自創的 JS 資料結構，框架 shader 不認識。R2 採用框架標準做法：

- 靜態盒子（牆面、傢俱、門、插座等）→ JS 端 `addBox()` → DataTexture → shader BVH 遍歷
- 旋轉物件（喇叭、腳架等有 rotY 的物件）→ 每個各一個 `uniform mat4 uXxxInvMatrix`，JS 端用 `THREE.Mesh` 的 `matrixWorld.invert()` 傳入，shader 內先把射線轉換到物件空間再做 AABB 測試（與 Cornell Box 範例完全相同的做法）
- 圓柱體（ISO-PUCK）→ `uniform vec3 uPuckPositions[8]` + `uPuckRadius` + `uPuckHalfH`，shader 自訂圓柱交叉測試

SOP 內的座標資料全部來自舊專案，容器格式改為框架標準，幾何數值不變。

### R2-13 ✅ X-ray 透視剝離（附著啟發式剔除）

**目標檔案**：
- `/Users/eajrockmacmini/Documents/VS Code/My Project/Home_Studio_3D/shaders/Home_Studio_Fragment.glsl`
- `/Users/eajrockmacmini/Documents/VS Code/My Project/Home_Studio_3D/js/Home_Studio.js`

**背景**：R2-3（commit 23636be）將 shader 自 `BoxInteriorIntersect` 改為 16 獨立 Box 之 `BoxIntersect`，因此原框架「相機出箱自動透視」之行為失效。Cam 1 於 z=3.310（南牆外）等屋外視角將被實心牆遮蔽。

**目標**：相機位於室內 → 全物件顯示，分毫不動；相機位於室外某軸向 → 該向牆及其附著物件（吸音板、衣櫃、樑柱、插座、軌道燈等）自 primary ray 剔除，視線穿透直達對向內壁。

#### 室內 AABB 判定

| 軸 | 下界 I_min | 上界 I_max | 說明 |
|----|-----------|-----------|------|
| X | -1.91 | 1.91 | 西牆內面至東牆內面 |
| Y | 0.0 | 2.905 | 地面至天花板 |
| Z | -1.874 | 3.056 | 北牆內面至南牆內面 |

**相機方位分類**（ε = 0.01m，避免 jitter）：

```
若 camPos.x > I_max.x + ε → 東向剝離啟動
若 camPos.x < I_min.x - ε → 西向剝離啟動
若 camPos.z > I_max.z + ε → 南向剝離啟動
若 camPos.z < I_min.z - ε → 北向剝離啟動
若 camPos.y > I_max.y + ε → 頂向剝離啟動（可選）
若 camPos.y < I_min.y - ε → 底向剝離啟動（可選）
```

相機同時觸發多軸時（如東南角屋外），對應軸向皆啟動，物件只要命中任一即剔除。

#### 附著啟發式判準

**閾值 T = 0.30m**（涵蓋牆厚 20cm、GIK 深 11.8cm、衣櫃貼牆面、樑柱 16cm 等；喇叭/插座皆不及此距離）。

對每一啟動之剝離方向，逐一檢視 Box / rotated mesh / cylinder 之 AABB：

```
東向剝離：box.maxCorner.x > I_max.x - T    → 判為東牆系 → 剔除
西向剝離：box.minCorner.x < I_min.x + T    → 判為西牆系 → 剔除
南向剝離：box.maxCorner.z > I_max.z - T    → 判為南牆系 → 剔除
北向剝離：box.minCorner.z < I_min.z + T    → 判為北牆系 → 剔除
頂向剝離：box.maxCorner.y > I_max.y - T    → 判為天花板系 → 剔除（含吸頂燈）
底向剝離：box.minCorner.y < I_min.y + T    → 判為地板系 → 剔除
```

| 東牆 GIK（E1-E3） | [1.792, 1.91] | 1.91 | ✓ | 剔除 |
| 東牆衣櫃 | [1.35, 1.91] | 1.91 | ✓ | 剔除 |

| 工作桌 | [-0.60, 0.60] | 0.60 | ✗ | 保留 |

**旋轉物件（KH150、腳架）之判準**：旋轉後 AABB 須重新包絡計算。最簡：取世界座標中心 ± 最大半尺寸對角線長度，作為保守 AABB。

**圓柱物件（ISO-PUCK、吸頂燈）**：以中心 ± radius 組 AABB。

#### 套用範圍：僅 primary ray

```
if (bounces == 0 && isBoxCulled(i)) continue;
```

**僅主視線穿透**，二次反彈、陰影射線、diffuse bounce 皆視該物件為實體。

**理由**：
1. 室內光場不變（牆面反射、GIK 吸音、樑柱遮光皆保留）
2. 避免「看起來空洞但地面仍有牆影」之詭異視覺
3. 業界 X-ray rendering 之標準做法
