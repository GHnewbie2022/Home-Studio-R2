**核心：三個 JS 函式**

函式 1（搬自舊專案 `Path Tracking 260412a 5.4 Clarity.html:1752`）：
```javascript
function lumenToCandela(lm, beamAngleDeg) {
    const halfRad = Math.max(0.01, (beamAngleDeg / 2.0)) * Math.PI / 180.0;
    return lm / (2.0 * Math.PI * (1.0 - Math.cos(halfRad)));
}
```

函式 2（新增，candela → radiance）：
```javascript
function candelaToRadiance(cd, emitterAreaM2) {
    // 假設 emitter 面發光均勻、Lambertian 外放
    // L = I / A（對 spot emitter 近似；真正 Lambertian 需乘 π 但此處以 emitter 面積分攤）
    return cd / Math.max(emitterAreaM2, 1e-6);
}
```

函式 3（新增，photopic 光通量 → 輻射通量）：
```javascript
function lumensToWatts(lm, kelvin) {
    // 依色溫查光視效能 K(T)
    //   3000K: 約 300 lm/W
    //   4000K: 約 320 lm/W
    //   6500K: 約 340 lm/W
    const K = kelvinToLuminousEfficacy(kelvin);  // 新表查
    return lm / K;
}
```

## R3-2 色溫 → RGB 換算 ⬜

**目標檔案**：

**演算法選擇**：Tanner Helland 近似（速度快、誤差 < 5%、對攝影棚色溫範圍 1000K~10000K 適用）。

```javascript
function kelvinToRGB(kelvin) {
    const t = Math.max(1000, Math.min(40000, kelvin)) / 100;
    let r, g, b;

    // Red
    if (t <= 66) r = 255;
    else r = 329.698727446 * Math.pow(t - 60, -0.1332047592);

    // Green
    if (t <= 66) g = 99.4708025861 * Math.log(t) - 161.1195681661;
    else g = 288.1221695283 * Math.pow(t - 60, -0.0755148492);

    // Blue
    if (t >= 66) b = 255;
    else if (t <= 19) b = 0;
    else b = 138.5177312231 * Math.log(t - 10) - 305.0447927307;

    return [
        Math.max(0, Math.min(255, r)) / 255,
        Math.max(0, Math.min(255, g)) / 255,
        Math.max(0, Math.min(255, b)) / 255
    ];
}
```

**光視效能查表**（lumensToWatts 用）：

```javascript
function kelvinToLuminousEfficacy(kelvin) {
    // 基於 Planck 黑體 × CIE photopic curve 積分（簡表）
    // 可進一步查表或用 1-D 插值
    if (kelvin <= 2700) return 280;
    if (kelvin <= 3000) return 300;
    if (kelvin <= 4000) return 320;
    if (kelvin <= 5000) return 330;
    if (kelvin <= 6500) return 340;
    return 350;
}
```

**Part A：廣角燈 emission 接入**

- `uTrackWideBeamCos` = vec2(cos(55°), cos(60°))（固定值）
- emitter 面積更大，candela 換算後 radiance 較 TRACK 低，但總流明高

**Part B：Many-Light Sampling（關鍵技術）**

目前（R3-4 末）NEE 為「每個 diffuse bounce 對每盞燈各做一次 shadow ray」。光源數量達 10 盞（4 Cloud + 4 Track + 2 Wide）時，shadow ray 成本 × 10 會拖慢 fps。

做法：每次 NEE 只選 1 盞燈，stochastic 均勻抽 1 of 10，selectPdf = 1/10。

```glsl
int NUM_LIGHTS_TOTAL = 10;  // R3 完整狀態
int selectedIdx = int(rand() * float(NUM_LIGHTS_TOTAL));
float selectPdf = 1.0 / float(NUM_LIGHTS_TOTAL);

// 依 selectedIdx 分派到 Cloud / Track / TrackWide sampling 子函式
if (selectedIdx < 4) sampleCloudLight(selectedIdx, ...);
else if (selectedIdx < 8) sampleTrackLight(selectedIdx - 4, ...);
else sampleTrackWideLight(selectedIdx - 8, ...);
```

**Part C：MIS Power Heuristic（β = 2）**

每個 diffuse bounce 做兩條 shadow ray：

```
1. NEE sample：對選中的光源做採樣，得 pdf_nee
2. BRDF sample：對 BRDF cosine-weighted 取樣，若命中光源則得 pdf_brdf
3. MIS 權重（power heuristic）：
   w_nee  = pdf_nee²  / (pdf_nee² + pdf_brdf²)
   w_brdf = pdf_brdf² / (pdf_nee² + pdf_brdf²)
4. 最終 contribution = w_nee * L_nee + w_brdf * L_brdf
```

對光滑表面（低 roughness）BRDF sample 收斂快；對粗糙表面 NEE 收斂快。MIS 自動切換。

**完成條件**：
1. ⬜ 2 盞廣角燈照亮房間中央部分，無光束邊緣
2. ⬜ Many-Light 實測：fps 較 R3-4 下降不超過 15%（單盞 NEE 成本不變，總 shadow ray 數減少 10 倍）
3. ⬜ MIS 上線後，粗糙表面（牆 0.9）variance 不劣於純 NEE，光滑金屬表面（Stand 支柱 0.2）variance 降低 3 倍以上

5. ⬜ Config 3 下開啟全部 10 盞燈，畫面亮度分佈均勻，無死角

## R3-6 indirectMul 校準歸一 ⬜

**目標檔案**：
- `/Users/eajrockmacmini/Documents/VS Code/My Project/Home_Studio_3D/js/Home_Studio.js`
- `/Users/eajrockmacmini/Documents/VS Code/My Project/Home_Studio_3D/shaders/Home_Studio_Fragment.glsl`

**前置條件**：R3-5 完成 ✅。

**背景**：
R2-18 `uIndirectMultiplier = 1.7` 是 Phase 2 框架 2-bounce truncation 下的肉眼補償。R3-5 真光源 + MIS 上線後，理論上提高 max_bounces 能自然收斂到正確能量，不必靠 1.7。

**做法**：

```
Step 1：max_bounces 從 4 提到 6
  → Cam 1/2/3 × 2000 spp 截圖（indirectMul=1.7 / 1.5 / 1.3 / 1.0 各一輪共 12 張）
  → 比對各級平均亮度分佈

Step 2：max_bounces 從 6 提到 8
  → 同樣四級 indirectMul，12 張截圖

Step 3：決定最終組合
  → 若 max_bounces=8 + indirectMul=1.0 與 R2-18 (bounces=4, mul=1.7) 亮度分佈接近（± 5%），則歸一成功
  → 若仍偏暗，保留 indirectMul 並記錄最終值
```

**R3-0 魔數同步處理**：
若 indirectMul 可歸 1.0，順便檢查 uLegacyGain 是否也可歸 1.0（兩者可能有部分耦合）。

**完成條件**：
1. ⬜ max_bounces 從 4 提到 8，fps 降幅不超過 30%
2. ⬜ 與 R2-18 定案版（900 + 1.7）對比亮度差異量化記錄於 Debug_Log
3. ⬜ 若 indirectMul 歸 1.0，GUI 將此 slider 標示「R3 歸一」；否則保留並註記
4. ⬜ 若 uLegacyGain 歸 1.0，移除 10 處 uniform 呼叫並簡化 shader
