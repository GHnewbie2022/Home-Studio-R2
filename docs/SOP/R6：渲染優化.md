## 主線 2：BVH 加速 ⏳ 進行中

### 動機

BVH 已是 LIVE 程式碼，但結構優化從未做。1024-spp 收斂秒數直接影響廠商討論節奏：

- 廠商來看 → 等 1024-spp 才看清楚 → 卡對話節奏
- BVH 結構優化 → 每 frame fragment shader 求交更快 → 收斂秒數縮短
- 對「採購決策可信度」+「廠商溝通即時性」直接 ROI

### R6-2 階段 1.5：分流優化（視盤點結果決定，2~5 天）

```
分流規則：

若瓶頸在 BVH traversal（> 30%）→ 結構優化
  a) 切到 SAH_Builder（如果現在不是）— 表面積啟發式分割命中率高
  b) BVH node packing 優化（記憶體存取模式）
  c) Stack-based vs while-loop traversal 切換評估

若瓶頸在 Light NEE（> 30%）→ 改 NEE 邏輯
  R3-6 已做 Many-Light Sampling + MIS，可能還有 cache 優化空間

若瓶頸在 Material BRDF（> 30%）→ shader 簡化
  分支扁平化、material switch 表化

若每段都 < 20%（已最佳化）→ R6-2 結案，不必硬優化
```

## 主線 1：後處理降噪（撤回 1 條、暫緩 2 條）

### 階段索引

```
階段 1：雙邊濾波後處理降噪 ❌ WITHDRAWN 2026-04-26
  狀態：ralplan 三輪共識 APPROVED → ultrawork 實作 7 commit → 實機驗證撤回
  撤回原因：5×5 cross-bilateral 對 path tracing 雜點本質是 spatial blur 非 denoise
           （對均勻雜點只變糊不變乾淨；對螢火蟲 weight=0 反而保留）

  物理機制不同於階段 1（時序面 vs 空間面），R6-1 撤回不否定本階段
  評估時機：主線 2 R6-2 完工後若 1024-spp 收斂仍嫌慢
  工時估：1~2 週

階段 3：OIDN AI 降噪 ⏸ 暫緩（長期投資）
  接 NVIDIA / Intel Open Image Denoise，需 WASM backend
  評估時機：階段 2 完成後若仍不滿意才上
  工時估：未估
```
