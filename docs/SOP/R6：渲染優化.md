## 主線 3：視覺收斂優化 ⏳ ← 你在這

> 動機：使用者實測 C3 / C4 切換後初期底噪過大、被迫只能用固定機位
> 目標：C3 / C4 從 0 spp 開始 30 ~ 60 秒內收到肉眼可接受
> 範圍：純後處理 / 時序累積 / 採樣權重微調；不動 BVH 結構、不動材質、不動光源端

### 動機（為什麼要開這條主線）

```
痛點實況
  C1 / C2 切換後初期收斂尚可
  C3 / C4 切換後初期底噪過大、收斂明顯比 C1 / C2 慢
  使用者「固定機位 C1 ~ C4」決策、是因為移動 / 切換時雜點太大
  固定機位這個決定、源自痛點本身、並非設計偏好

R6 主線 1 已榨乾肉眼後處理可救範圍
R6 主線 2 已逼近 GPU 速度上限
→ 只剩「per-spp 收斂率」這條路：每一個 spp 帶的雜訊量降下來
→ 跟 R6-1 撤回的雙邊濾波物理機制不同（時空累積 vs 純空間濾波）
```

### Phase 0：收斂瓶頸盤點（必前置）

```
動作      在 Brave DevTools 跑 GPU profile、量 C3 / C4 vs C1 / C2 各段時間
量測      1. fragment shader 內各段（光源直接採樣 / 反射計算 / BVH 走訪）耗時佔比
          2. 「初期 0 ~ 64 spp」階段每像素變異量
          3. C3 / C4 vs C1 / C2 在「光源挑樣」這步的命中分佈
工時      1 ~ 3 hr
產出      一份盤點報告、結論：
            路徑 A（光源端變異）    →  階段 2 SVGF
            路徑 B（反射計算端變異） →  階段 3 reproject + 路徑導引
            路徑 C（後處理端可救）  →  階段 4 OIDN 重評
守門      禁止未盤點直接動 shader（避免重蹈 R6-1 / R6-2 不盤點覆轍）
```

### 階段 2：SVGF（時空變異引導濾波）

```
做什麼（白話）
  把上一 frame 算過的結果、依「這個像素移動了多少」接過來
  接過來之後再用「這像素自己變異多大」當權重做空間平滑
  變異大的多平滑、變異小的少動

跟 R6-1 撤回的雙邊濾波差別
  R6-1 是死板的純空間濾波、對均勻雜點只變糊
  SVGF 多了「時序累積」+「依變異智慧權重」、為路徑追蹤雜點專門設計

工時估    1 ~ 2 週
風險      物件 / 視角移動會出現拖影、要加歷史拒絕邏輯
互動      跟 J3 暗角借光雙緩衝會撞、buffer pipeline 要重新設計
```

### 階段 3：時序重投影 + 路徑導引（reproject + path guiding）

```
做什麼（白話）
  reproject     相機移動中、上 frame 結果接過來複用、移動中雜點明顯改善
  path guiding  學習場景哪裡是亮的、把光線往「重要方向」打、不是亂打
  兩個合起來    移動中收斂率提升 + 初期 spp 雜點少

工時估    2 ~ 3 週（path guiding 是論文級獨立大題）
風險      跟 SVGF 部分功能重疊、需先決定走哪條
```

### 階段 4：OIDN 重評（Intel AI 降噪、瀏覽器版社群套件）

```
做什麼（白話）
  Intel 出的 AI 降噪函式庫、業界 offline 算圖標準
  目前社群有 oidn-web / oidn.js 之類瀏覽器版套件、效能跟維護狀態未實測

要先評估
  1. 2025 ~ 2026 社群版本是否堪用（效能、穩定度、API 對齊度）
  2. 從 WebGL framebuffer 讀回 → 餵 AI → 寫回 canvas 的每 frame I/O 成本
  3. AI 補出來的雜點是「猜的」、跟「真實光路示範」這個專案定位是否合適

工時估    社群版可直接整合：1 ~ 2 週
          要自己 port：跨入 R7 後續
風險      社群套件維護不穩、版本可能跟不上官方
備案      若瀏覽器版不堪用、整個專案改寫成 Mac 獨立 app（≥ 1 個月、屬下世代專案）
```

### 守門禁忌

```
1. 嚴禁未做 Phase 0 盤點直接動 shader（避免重蹈 R6-1 / R6-2 覆轍）
2. 嚴禁 1024 spp 收斂視覺對 main 出現偏差（拖影 = 直接回滾）
3. 嚴禁趁機改 NEE / 材質 / 光源端（屬 R7 範疇、需另開 ralplan）
4. 嚴禁拿 R6-1 撤回教訓覆蓋 R6-3（物理機制不同、不適用）
```

### 觸發 R7 條件（往上跳）

```
若 Phase 0 盤點顯示：
  瓶頸不在後處理 / 時序累積 / 採樣權重、而在「採樣演算法本身」
```
