## 候選方向（大綱、無排序、待 R6-3 結果決定優先序）

### 方向甲：ReSTIR（多光源時空複用採樣）

```
做什麼（白話）
  現在每個像素每 frame 都重新挑一盞光源算亮度貢獻
  挑光源這步運氣不好就會出現雜點
  ReSTIR 改成「每像素留一個小水塘、記住目前為止挑過的最好光源樣本」
  跟周圍像素 + 上一 frame 的水塘合併
  等於「自動學會哪盞光對這個像素重要」、不再亂挑

為什麼適合本專案
  C3 = 只開 Cloud 漫射燈（4 條燈條、4-rod 多光源）
  C4 = 只開軌道燈 + 廣角燈（Cloud 關閉）
  C3 與 C4 採樣特性不同（漫射 vs 方向性直接光）、ReSTIR 對多光源場景的挑樣變異是業界新標準

工時估    3~5 週（NVIDIA 2020 之後新標準、論文級實作）
風險      會動到 R3-6 既有多重重要性採樣（MIS）邏輯、改動範圍大
參考      Bitterli 2020 ReSTIR 原論文 + 後續 ReSTIR GI / Restir PT 衍生
```

### 方向乙：path guiding 重做（Müller 2017 級）

```
做什麼（白話）
  現在反彈光線的方向、是依「物件表面的反射特性（材質）」決定
  path guiding 多加一層學習：「這個位置的光線、過往哪些方向比較容易接到光源」
  把這份學習記錄起來、下次抽方向時偏向那些方向、雜點降很多

為什麼適合本專案
  暗角 / 遠離光源區域、間接光彈跳變異大
  path guiding 對「室內 + 多次反彈」場景效益顯著

工時估    4~6 週（含學習結構設計 + GPU 友善資料結構）
風險      跟 R6-3 階段 3 的 path guiding 部分重疊、需確認介接層
參考      Müller 2017 Practical Path Guiding + 後續 NN-based 衍生
```

### 方向丙：光源 importance sampling 機率優化（既有 R3-6.5 延伸）

```
做什麼（白話）
  既有 R3-6.5「Many-Light Sampling 動態 NEE pool」已是 stochastic light selection
  R7 重新審視 pool 的機率分佈、是否能依「光源到像素的距離 / 立體角」做更好的權重
  屬「在既有架構內改進」、不像 ReSTIR 是換架構

工時估    1~2 週（範圍最小、ROI 中等）
風險      動到 R3-6 既有 MIS、需與既有平衡點對齊
適合     若 R6-3 Phase 0 顯示「挑樣變異中等、不到 ReSTIR 級才能解」
```

### 方向丁：blue noise sampling 升級

```
做什麼（白話）
  現在 RNG（隨機數產生器）已用 IQ Shadertoy hash + Jacco Bikker
  但 blue noise 的時序分佈仍有調整空間
  例如改用 Heitz & Belcour 2019 的「Spatiotemporal Blue Noise」

為什麼考慮
  低 spp 階段（0~64 spp）肉眼觀感、blue noise 分佈品質直接影響「看起來舒不舒服」

工時估    3~5 天（範圍最小、屬 polish 級）
風險      低、屬可逆替換
適合     R7 收尾階段、其他大方向落定後的 polish
```

## 範圍邊界（守門禁忌）

```

2. R7 啟動必先 ralplan 共識

   既有「Cloud rod / 軌道燈 / 4 燈條」C1~C4 視覺結果不可走樣

4. R7 屬「大階段」、合併到 main 必依 R 主線 DONE 紀律
   R7-? 子階段完工屬本地 commit、整段 R7 完工才合併 main + push

5. 嚴禁把 R7 當 R6-3 的 fallback 投錢
   若 R6-3 已能達 30~60 秒目標、R7 不啟動、屬下世代專案範疇
```

## R6-3 → R7 銜接決策樹

```
R6-3 Phase 0 盤點完成
   ↓
路徑 A 多光源變異（C3/C4 主因是挑光源不準）
   → R6-3 階段 2 SVGF 試一輪
   → 若仍達不到 30~60 秒目標、R7 啟動、優先方向甲（ReSTIR）

路徑 B 反射計算端變異（暗角間接光彈跳變異大）
   → R6-3 階段 3 reproject + path guiding 試一輪
   → 若仍達不到目標、R7 啟動、優先方向乙（path guiding 重做）

路徑 C 後處理可救（社群 OIDN web port 堪用）
   → R6-3 階段 4 OIDN 整合
   → 若達標、R7 不啟動

複合路徑（A + B 同時觸發）
   → R6-3 兩條同跑、依 ROI 決定先後
   → 若仍達不到目標、R7 啟動、方向甲 + 方向乙 兩階段串接

全部不打中
   → R7 全方向重評、可能需引入方向丙（importance sampling 優化）作中介

明確決策節點
   每個 R7 候選方向啟動前、必對 R6-3 階段對應結果做事後驗證、避免重複投資
```
