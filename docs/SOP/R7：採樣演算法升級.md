```
2026-05-07 v1 實作：R7-2 light importance sampling v1

  新增 console helper：
    setR72QuickPreviewIsolation(true / false)
    reportR72QuickPreviewIsolationConfig()
  setR72QuickPreviewIsolation(true) 會：
    切到 C3。
    切到快速預覽。

    把 uBorrowStrength 設為 0。

    暗角借光不是主要遮蔽源。

    後續不要把「趨近真實 1 SPP 變乾淨」誤當成「快速移動預覽已解」。

  InitCommon: js/InitCommon.js?v=r7-2-light-importance-sampling-v1-r7-2d-step-history
  Home_Studio: js/Home_Studio.js?v=r7-2-light-importance-sampling-v1-r7-2d-step-history
```

```
  v3r 依使用者要求，把同一條固定曲線擴到 C4 快速預覽試驗。
  v3s 依使用者要求，新增 C3/C4 同條件 GIK vs wall 低 SPP 亮度量測。
  v3t 依量測結果，新增 C4-only GIK dark-only lift，針對低亮度 GIK 額外補，亮區用 cap 壓住。

  v3v 依使用者肉眼回報，繼續讓 C4 牆面更暗，C4 GIK dark-only lift 更亮。
  v3al 依使用者修正，C4 回到沒有 R7-3 terminal fill 的狀態，並保留第一個可見畫格直接到 2SPP。

目前結論
  R7-3 的 C3 固定曲線已收尾；C4 試驗路線已退回原本狀態。
  曲線調整已收斂到 v3o。
  v3p 拔掉 R7-3 左下 UI，改為預設 ON。
  v3p 套用範圍改成 C3-only，C4 不套用 R7-3 terminal fill。
  v3q 只把 ON/OFF 開關做回左下，不恢復曲線輸入。
  v3r 套用範圍改成 C3 / C4；C1 / C2 不套用。
  v3al 已把 R7-3 terminal fill 套用範圍改回 C3-only，C4 保留丟可見 1SPP。
  預設仍為 ON；固定曲線不變。
  使用者肉眼回報目前差不多，可以暫時收曲線。
  2026-05-08 高 SPP 驗收：使用者測 1000SPP，R7-3 ON / OFF 看起來完全一樣。
  判定：lowSppFade 高 SPP 守門通過；R7-3 不影響正式高 SPP 畫面。
  2026-05-08 追加驗收：C3 丟掉可見 1SPP、第一眼直接到 2SPP 的手感可接受。
  判斷理由：1SPP 最髒，2SPP 與 3SPP 已比較接近；目前 FPS 本來就不高，只丟掉 1SPP 對操作手感影響小。
  2026-05-08 追加試驗：C4 快速預覽也套用同一條 R7-3 固定曲線，且第一個可見畫格直接到 2SPP。
```

### R7-4：ReSTIR（多光源時空複用採樣，暫緩）

```
做什麼（白話）
  現在每個像素每 frame 都重新挑一盞光源算亮度貢獻
  挑光源這步運氣不好就會出現雜點
  ReSTIR 改成「每像素留一個小水塘、記住目前為止挑過的最好光源樣本」
  跟周圍像素 + 上一 frame 的水塘合併
```
