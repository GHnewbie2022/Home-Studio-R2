# R7-3.8 C1 1000SPP Bake Capture Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Correctly capture C1 1000SPP baked-lighting reference data without visible diagnostic UI, using the raw HDR accumulation buffer first and direct surface texels for the actual bake path.

**Architecture:** Keep normal rendering unchanged by default. Add programmatic capture helpers that switch to C1, wait for a target sample count, read float render-target data, record surface classification, then run a direct surface-texel atlas pass at the requested bake resolution. Codex runs the capture through an automation script; the user should not need to operate browser DevTools or paste console commands.

**Tech Stack:** Vanilla JS, Three.js/WebGL2, existing path-tracing shader pipeline, FloatType WebGLRenderTarget, programmatic capture helpers, Node contract tests, Node/CDP capture runner.

- MDN `readPixels()` reads a rectangle from the current color framebuffer into a `TypedArray` or `DataView`; `gl.FLOAT` maps to `Float32Array`.
  - Source: https://developer.mozilla.org/en-US/docs/Web/API/WebGLRenderingContext/readPixels

## Data Specification And Execution SOP

This plan produces a C1 bake reference package. A package is valid only when the manifest, raw HDR reference, direct atlas pixels, texel metadata, validation report, and artifact paths agree with each other.

### 1. C1 Bake Data Definition

C1 bake data has three layers:

```text
1. Raw HDR reference
   - Source: screenCopyRenderTarget.
   - Purpose: camera-view reference for checking the current path-traced C1 result.
   - Sample target: at least 1000SPP.
   - Format: Float32 RGBA.
   - Display processing: excluded.

2. Surface classification reference
   - Source: first-visible surface class probe.
   - Purpose: confirm which screen pixels correspond to floor, GIK, ceiling, wall, object, or background.
   - Format: class histogram plus optional class mask.

3. Direct atlas bake data
   - Source: direct surface texels.
   - Purpose: actual bake-source data for static room surfaces.
   - Sample target: at least 1000SPP per texel.
   - Format: Float32 RGBA atlas plus per-texel metadata.
```

The first R7-3.8 package must include one direct floor-center patch and the full raw HDR reference. A later package may add more patches, but every included patch must follow the same contracts below.

### 2. Atlas Resolution Policy

The atlas output resolution is chosen before capture and must equal the actual render target and saved binary dimensions.

```text
Rules:
1. targetAtlasResolution is required in the report.
2. atlasSummary.patchSize must equal targetAtlasResolution.
3. atlasSummary.upscaled must be false.
4. report.upscaled must be false.
5. The runner must fail when saved atlas dimensions differ from targetAtlasResolution.
6. A low-resolution diagnostic atlas may exist only as a separate smoke test artifact with smokeTest: true.
7. A smoke test artifact is never accepted as the C1 bake source.
```

Default first patch:

```json
{
  "patchId": 0,
  "surfaceName": "floor_center_c1_reference",
  "surfaceClass": "floor",
  "targetAtlasResolution": 512,
  "samplesPerTexel": 1000,
  "upscaled": false
}
```

If the implementation selects 1024 or another final target, all three values must match: uniform resolution, render target size, and saved binary dimensions.

### 3. Texel Mapping Contract

Each atlas texel must map to a known static room surface point. The mapping must be deterministic and saved with the capture.

Create `docs/data/r7-3-8-c1-bake-surface-spec.json`:

```json

  "patches": [
    {
      "patchId": 0,
      "surfaceName": "floor_center_c1_reference",
      "surfaceClass": "floor",
      "resolution": { "width": 512, "height": 512 },
      "samplesPerTexel": 1000,
      "worldBounds": {
        "xMin": -1.0,
        "xMax": 1.0,
        "zMin": -1.0,
        "zMax": 1.0,
        "y": 0.01
      },
      "normal": [0.0, 1.0, 0.0],
      "includedInFirstPackage": true,
      "upscaled": false
    }
  ]
}
```

Per-texel metadata layout:

```text
texel-metadata-f32.bin
stride: 12 float32 values per texel
0: worldPosition.x
1: worldPosition.y
2: worldPosition.z
3: normal.x
4: normal.y
5: normal.z
6: surfaceClassId
7: materialId
8: objectId
9: patchId
10: texelU
11: texelV
```

Class IDs:

```text
0 background_or_invalid
1 floor
2 gik
3 ceiling
4 wall
5 object
```

The metadata bin must contain exactly `targetAtlasResolution * targetAtlasResolution * 12` float values for each patch.

### 4. 1000SPP Accumulation Contract

The raw HDR reference and direct atlas use separate sample counters in the report:

```text
rawHdr.actualSamples:
  sample count for screenCopyRenderTarget.

atlasSummary.actualSamples:
  sample count for the direct surface-texel atlas render target.
```

Atlas accumulation rules:

```text
1. Create an atlas render target whose width and height equal targetAtlasResolution.
2. Reset atlas accumulation before capture starts.
3. Render the same atlas patch until atlasSummary.actualSamples >= 1000.
4. Every texel in the patch receives the same number of atlas samples.
5. Store actualSamplesByPatch in the report.
6. Fail the runner when any included patch reports fewer than 1000 samples.
7. Restore normal render mode after capture in a finally block.
```

Required report field:

```js
actualSamplesByPatch: [
  { patchId: 0, surfaceName: 'floor_center_c1_reference', actualSamples: 1000 }
]
```

### 5. Validation SOP

The capture runner must perform these checks before it prints success:

```text
1. Version check
   - report.version === 'r7-3-8-c1-1000spp-bake-capture'
   - report.config === 1

2. Raw HDR check
   - rawHdr.actualSamples >= 1000
   - rawHdrSummary.nonFinitePixels === 0
   - rawHdrSummary.finitePixels === buffer.width * buffer.height

3. Surface class check
   - surfaceClassSummary.floor > 0
   - surfaceClassSummary.ceiling > 0
   - surfaceClassSummary.wall > 0
   - At least one valid static class exists in the captured view.

4. Atlas resolution check
   - atlasSummary.patchSize === report.targetAtlasResolution
   - report.upscaled === false
   - atlasSummary.upscaled === false
   - saved atlas float count === targetAtlasResolution * targetAtlasResolution * 4

5. Texel metadata check
   - saved metadata float count === targetAtlasResolution * targetAtlasResolution * 12
   - validTexelRatio >= 0.99 for included patches
   - normal length is between 0.99 and 1.01 for valid texels

6. Sample count check
   - atlasSummary.actualSamples >= 1000
   - every actualSamplesByPatch entry is >= 1000

7. Reprojection sanity check for first floor patch
   - Pick at least 64 deterministic texels from the patch.
   - Project their world positions back into the current camera.
   - Compare luma against matching raw HDR pixels when the surface class is floor.
   - Record medianRelativeLumaError and p90RelativeLumaError.
   - This is diagnostic only for R7-3.8, because camera-view floor radiance and direct atlas normal-direction radiance can differ on rough/specular surfaces.
   - Record a warning when medianRelativeLumaError > 0.25 or p90RelativeLumaError > 0.50.
```

The reprojection check is a sanity check, not an acceptance gate for R7-3.8. Reflection and directional radiance handling is outside this R7-3.8 capture package and must be recorded as a separate follow-up when reflective or view-dependent surfaces enter the atlas.

Add in `js/InitCommon.js`:

```js
async function readR738RenderTargetFloatPixels(renderTarget) {
  const width = renderTarget.width;
  const height = renderTarget.height;
  const pixels = new Float32Array(width * height * 4);
  if (typeof renderer.readRenderTargetPixelsAsync === 'function') {
    await renderer.readRenderTargetPixelsAsync(renderTarget, 0, 0, width, height, pixels);
  } else {
    renderer.readRenderTargetPixels(renderTarget, 0, 0, width, height, pixels);
  }
  return { width, height, pixels };
}
```

Add summary helper:

```js
function summarizeR738RawHdrPixels({ width, height, pixels }) {
  const lumas = [];
  let finitePixels = 0;
  let nonFinitePixels = 0;
  let r = 0;
  let g = 0;
  let b = 0;
  let maxLuma = 0;

  for (let i = 0; i < pixels.length; i += 4) {
    const pr = pixels[i];
    const pg = pixels[i + 1];
    const pb = pixels[i + 2];
    if (!Number.isFinite(pr) || !Number.isFinite(pg) || !Number.isFinite(pb)) {
      nonFinitePixels += 1;
      continue;
    }
    finitePixels += 1;
    r += pr;
    g += pg;
    b += pb;
    const luma = 0.2126 * pr + 0.7152 * pg + 0.0722 * pb;
    lumas.push(luma);
    maxLuma = Math.max(maxLuma, luma);
  }

  lumas.sort((a, b) => a - b);
  const pick = (q) => lumas.length ? lumas[Math.min(lumas.length - 1, Math.floor(q * (lumas.length - 1)))] : 0;

  return {
    width,
    height,
    finitePixels,
    nonFinitePixels,
    meanRgb: finitePixels ? [r / finitePixels, g / finitePixels, b / finitePixels] : [0, 0, 0],
    meanLuma: finitePixels ? lumas.reduce((sum, value) => sum + value, 0) / finitePixels : 0,
    p50Luma: pick(0.50),
    p90Luma: pick(0.90),
    p99Luma: pick(0.99),
    maxLuma
  };
}
```

The helper reads `screenCopyRenderTarget`, because it is the copied HDR accumulation before `ScreenOutput` display processing.

```js
window.waitForR738C1Samples = async function waitForR738C1Samples(targetSamples = 1000, timeoutMs = 180000) {
  const startedAt = performance.now();
  while (performance.now() - startedAt < timeoutMs) {
    const actualSamples = Number(sampleCounter || pathTracingUniforms?.uSampleCounter?.value || 0);
    if (actualSamples >= targetSamples) return actualSamples;
    if (typeof wakeRender === 'function') wakeRender('r7-3-8-c1-bake-wait');
    await new Promise((resolve) => setTimeout(resolve, 250));
  }
  throw new Error(`R7-3.8 C1 bake capture timeout before ${targetSamples} samples`);
};
```
