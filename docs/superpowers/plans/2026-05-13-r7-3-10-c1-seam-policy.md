## 官方文件依據

three.js `DataTexture` 官方文件寫明 `magFilter` 與 `minFilter` 預設為 `THREE.NearestFilter`。普通 `Texture` 官方文件寫明 `magFilter` 預設為 `LinearFilter`，`minFilter` 預設為 `LinearMipmapLinearFilter`。three.js texture manual 說明，低解析貼圖使用 `NearestFilter` 放大時會呈現像素化外觀。

參考：

```markdown
1.  https://threejs.org/docs/api/en/textures/DataTexture.html
2.  https://threejs.org/docs/pages/Texture.html
3.  https://threejs.org/manual/en/textures.html
```

## 統一資料來源

後續設計應建立兩張表，讓 JS 與 shader 共享同一套規則：

```javascript
const OCCLUDER_TABLE = Object.freeze([
  {
    id: 'wardrobe',
    kind: 'static_box',
    worldOOBB: {
      center: [1.63, 0.9775, -1.2885],
      halfExtents: [0.28, 0.9775, 0.5855],
      yaw: 0
    },
    contactSurfaces: ['floor', 'north_wall', 'east_wall']
  }
]);

const SURFACE_REGISTRY = Object.freeze({
  c1_floor_full_room: {
    targetId: 1001,
    plane: { axis: 'y', value: 0.01 },
    mapping: 'xz'
  },
  c1_north_wall: {
    targetId: 1002,
    plane: { axis: 'z', value: -1.874 },
    mapping: 'xy',
    doorHoles: ['north_door']
  },
  c1_east_wall: {
    targetId: 1003,
    plane: { axis: 'x', value: 1.91 },
    mapping: 'zy'
  }
});
```

規則：

```markdown
1.  同一個 occluder / surface 規則只能有一份來源。
2.  shader 端不可另寫一套常數判斷。
3.  contract test 要檢查來源資料與 shader uniform / generated code 一致。
```

## Bake 階段規則

```markdown
1.  每個 texel 先由 SURFACE_REGISTRY 轉成 world position。
2.  若 texel quad 與 OCCLUDER_TABLE 的靜態物件接觸區相交，該 texel alpha = 0。
3.  若 texel 落在門洞區，該 texel alpha = 0。
4.  alpha = 0 的 texel 不進行漫射烘焙。
5.  dilation 只允許最多 4 pixel 邊界填色。
6.  4 pixel 之外仍保持 alpha = 0。
7.  dilation 不可再用整張 atlas flood-fill。
```

## Runtime 規則

```glsl
bool r7310Lookup(hit, out vec3 bakedRadiance)
{
  SurfaceTag tag = classifyHit(hit);
  if (tag == NONE) return false;

  vec2 uv = surfaceToAtlasUv(tag, hit.worldPosition);
  vec4 sample = texture(tR7310C1FullRoomDiffuseAtlasTexture, uv);
  if (sample.a < 0.5) return false;

  bakedRadiance = sample.rgb;
  return true;
}
```

規則：

```markdown
1.  atlas alpha 是 runtime 的最終有效性訊號。
2.  alpha < 0.5 時，直接回到 live path tracing。
3.  runtime 不再另寫 hidden contact 判斷。
4.  門洞、家具接觸區、未處理區都由 alpha 表示。
```

## Filter 與解析度

下一輪需先做 feasibility test：

```markdown
1.  檢查 renderer.extensions.has('OES_texture_float_linear')。
2.  若支援 FloatType + LinearFilter，先走 FloatType + LinearFilter。
3.  若 FloatType 線性取樣不支援，評估 HalfFloatType + LinearFilter。
4.  任何 LinearFilter 路線都必須先完成 alpha policy。
5.  atlas 解析度目標先估 texel <= 5 mm；牆面與地板優先評估 1024。
```

## Shared Edge 規則

後續要把表面共邊當成一級驗收項目：

```markdown
1.  floor / north wall 共邊：z = -1.874。
2.  floor / east wall 共邊：x = 1.91。
3.  north wall / east wall 共邊：x = 1.91, z = -1.874。
4.  每條共邊取對應 texel pair。
5.  雙方都 valid 時，相對 luma 差應低於 10%。
6.  一方 alpha = 0 時，不可用另一方數值硬補。
```

## 必補測試

```markdown
1.  seam continuity test：
    shared edge texel pair 的 relative luma diff < 0.10。
2.  dilation contamination test：
    dilation 區與附近 valid 區的亮度跳變超過門檻時 fail。
3.  reprojection gate：
    reprojectionStatus = fail 或 comparisons = 0 時，runner status 必須 fail。
4.  alpha fall-through test：
    alpha = 0 的 texel 不能 short-circuit。
5.  atlas filter feasibility test：
    記錄 FloatType / HalfFloatType 的 linear filter 支援狀態。
6.  visual diff harness：
    同視角 atlas-on / atlas-off ROI 量測接縫差異。
```
